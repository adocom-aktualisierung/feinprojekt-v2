01 · Executive Summary

Kernergebnisse auf einen Blick

Die Seite ist insgesamt auf einem sehr soliden Niveau — Skip-Link, Focus-Ring, Mobile-Nav-Focus-Trap, Form-Error-Summary, reduced-motion und i18n-Key-Coverage sind vorbildlich umgesetzt. Die gefundenen Probleme betreffen primär i18n-Lücken an Primär-Inhalten (H1, Bilder), Touch-Targets unterhalb Senior-Minimum 48×48 und fehlende ARIA-Pattern am Filter.

Top-3 FEIN-Review-Blocker

  1. F1 — Hardcoded H1 auf Startseite: „Gemeinsam Kochen – Gemeinsam Wachsen" hat kein data-i18n-Attribut. Englischsprachige Nutzer:innen sehen den deutschen Titel.
  2. F2 — Dialog-Close zu klein: 40×40 px — verletzt WCAG 2.5.5 AAA (44×44) und das Projekt-Minimum (48×48) für Senior:innen-Zielgruppe.
  3. F3 — Filter-Chips ohne ARIA-Pattern: Keine aria-pressed, keine Ergebnis-Announce für Screen-Reader, kein Empty-State bei Null Treffern.

Empfehlung: Alle P0-Befunde vor dem FEIN-Review beheben (~0,5 Personentage). P1 idealerweise ebenfalls — die Senior:innen-Touch-Targets und i18n-Lücken an Workshop-Metadaten wirken sich direkt auf das Nutzungserlebnis der Primärzielgruppe aus.

02 · Methodik

Werkzeuge & Bewertungsraster

Live-Verifikation

Alle Befunde wurden am laufenden Vite-Dev-Server (localhost:5173) live verifiziert — nicht nur statisch aus dem Code abgeleitet. Methoden:

  • Computed Styles via preview_inspect (Touch-Targets, Kontraste, Fokus-Ring) auf Selektoren wie .btn-primary, .dialog-close, .filter-chip, .contact-bar-item
  • DOM-Snapshots via preview_eval für i18n-Key-Coverage-Check aller 10 Hauptseiten gegen public/locales/en.json
  • Interaktions-Tests: Dialog showModal() + Initial-Focus-Check, Filter-Click + Hidden-State-Check
  • Viewport-Switching: Mobile 375×812, Desktop 1440×900
  • Konsolen-Logs geprüft auf JS-Fehler (nur Vite-HMR-Messages, keine Applikations-Fehler)

Fix — js/filter.js
export function initFilters() {
  document.querySelectorAll('.filter-bar').forEach(bar => {
    const chips = bar.querySelectorAll('.filter-chip');
    const container = document.getElementById(bar.dataset.filterGroup);
    if (!container) return;
    const items = container.querySelectorAll('[data-filter-target]');
    const status = document.getElementById('filter-status');
    const emptyState = document.getElementById('filter-empty');

    chips.forEach(chip => {
      chip.addEventListener('click', () => {
        const filter = chip.dataset.filter;
        chips.forEach(c => {
          c.classList.toggle('is-active', c === chip);
          c.setAttribute('aria-pressed', c === chip ? 'true' : 'false');
        });
        let visible = 0;
        items.forEach(item => {
          const match = filter === 'all' || item.dataset.category === filter;
          item.classList.toggle('is-hidden', !match);
          if (match) visible++;
        });
        if (emptyState) emptyState.hidden = visible !== 0;
        if (status) {
          status.textContent = visible === 1
            ? '1 Workshop gefunden'
            : `${visible} Workshops gefunden`;
        }
      });
    });

    // Reset-Button im Empty-State
    document.querySelector('[data-clear-filter]')?.addEventListener('click', () => {
      bar.querySelector('.filter-chip[data-filter="all"]').click();
    });
  });
}
Bonus: Arrow-Key-Navigation (Toolbar-Pattern, WAI-ARIA APG)
// Optional aber empfohlen für P0-Fix
bar.addEventListener('keydown', (e) => {
  if (!['ArrowLeft','ArrowRight','Home','End'].includes(e.key)) return;
  const arr = [...chips];
  const idx = arr.indexOf(document.activeElement);
  if (idx === -1) return;
  e.preventDefault();
  let next = idx;
  if (e.key === 'ArrowRight') next = (idx + 1) % arr.length;
  if (e.key === 'ArrowLeft')  next = (idx - 1 + arr.length) % arr.length;
  if (e.key === 'Home') next = 0;
  if (e.key === 'End')  next = arr.length - 1;
  arr[next].focus();
});

Font-Size-Switcher wirkt nicht auf Above-Header-Schrift

F22 P2 CLAUDE.md (Senior)
Dateien
css/tokens.css:150-156, css/layout.css:46, 68, 105, 136
IST

Fix (zwei Varianten)
// Variante A: Above-Header auf Token umstellen
.above-header .header-phone { font-size: var(--font-size-small); }

// Variante B: Eigenes Token für Utility-Strip
:root { --font-size-utility: 0.8125rem; }
html[data-font-size="large"] { --font-size-utility: 1rem; }
.above-header … { font-size: var(--font-size-utility); }

Impact: Senior:innen, die „A+" aktivieren, erwarten Vergrößerung überall. Utility-Strip bleibt klein.

Fehlender Fallback für .fade-up ohne JS

F23 P2 WCAG 1.4.12
Datei
css/animations.css:14-24
IST
.fade-up-Elemente starten mit opacity: 0. Wenn js/main.js nicht lädt (z.B. CSP-Fehler, Script-Block), bleiben sie unsichtbar. IntersectionObserver fügt .is-visible hinzu — ohne JS kein .is-visible.
Fix (<html class="no-js">-Pattern)
// Inline im <head> vor Stylesheets:
<script>document.documentElement.classList.remove('no-js');
         document.documentElement.classList.add('js');</script>

// css/animations.css
.no-js .fade-up { opacity: 1; transform: none; }

Registrierungs-Dialog — expliziter Focus-Trap fehlt

Datei

IST
Native <dialog>-Element + showModal() trappt Focus in modernen Browsern automatisch (top-layer). In älteren Versionen oder bei Polyfills fehlt der Trap. Das Mobile-Nav-Pattern aus js/main.js:10-61 lässt sich wiederverwenden.
